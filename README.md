The POP ontology aims at describing data from different perspectives, namely:
* Prescriptions, such as specifications, mandatory standards, etc.
* Observations; here, SOSA/SSN rules;
* Predictions,
* Corrections (new prescriptions), as may result from maintenance processes,

and to enable their expression and comparison "inside the knowledge graph", allowing data from heterogenous sources to be documented in view of further processing, and avoiding risks of semantic misalignments.

For instance, prescribed toleranced values shall be compared to observed values with uncertainties and predicted predicted values with confidence intervals. The POP ontology provides a knowledge base for data reconciliation, or for prediction model re-calibration.
